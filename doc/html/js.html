Le Fonctionnement des Scripts JS

Il y a plusieurs parties dans notre utilisation du Javascript, car pour les affichages des UMLs, nous utilisons l'API Joint Js, qui est codé en Javascript. Nous avons donc une partie du code qui est généré selon la syntaxe permettant de faire des UMLs avec cette API, et d'un autre côté, nous avons du complementer cette API.

JointJS

JointJS est une API de modélisation en Javascript, c'est la version open source de Rapid, une version payante mais beaucoup plus complète que Joint js.
Nous avons décidé de travailler avec cette API, plutôt qu'avec le logiciel/site Plant uml. En effet PlantUML génère une image en fonction du code donné, et donc l'UML obtenu est static et ne peut pas etre modifié. Grâce à Joint Js, l'affichage devient interactif et cela permet souvent de gagner en lisibilité sur le code

Pour une question de portabilité et aussi pour une question de stockage, nous avons stocké tous les scripts de JointJS sur un VPS distant, et nous avons donc plus qu'un fichier JS a générer en local

Le choix d'afficher certaines relations et non d'autres, notamment les agrégations, est un choix stratégique car pour détecter et représenter tous les agrégations/compositions d'un code, il faut un code détecteur et un analyseur syntaxique beaucoup plus développer . Nous avons donc choisi de laisser de coté cette parti pour uniquement afficher les relations d'héritages et celle d'association (lorsqu'une classe utilise une autre ... ). Mais le code a été fait en POO, et de façon modulaire. Ainsi, plus tard nous pourrions ajouter ces parties manquantes au code. Le problème principal de JointJS est que, c'est une API assez pauvre en matière de choix/possibilité. Nous avons donc dû coder des compléments.

Complement

Nous avons plusieurs compléments, stockés dans deux fichiers et stockés sur le VPS :

  • [complement.js]
  • [pattern.js]

complement.js est un script qui regroupe toutes les fonctions qui permet de nettoyer l'UML, de gérer la boîte à outils et de mettre en place les layouts.

pattern.js est un script JS contenant une classe pattern et qui permet de créer un layout évolué. Cette classe est conçue pour être mise à jour et ainsi donner un layout plus performant

Les scripts Javascripts sont assez rudimentaires car ce n'était pas la partie la plus importante du code, celle-ci se trouvant sur les scripts Python
