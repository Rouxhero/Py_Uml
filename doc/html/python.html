Le Fonctionnement des Scripts Python

Cette partie est dedié au code le plus important de notre projet.

Le fonctionnement python est divisé en deux parties, une partie dans le [main.py] et la suite dans le dossier [languages].

Sommaire

  • main.py
  • languages

Main.py

Ce script Python est le routeur qui fait le lien entre l'extension sur VScode et les différents langages pour traduire. Il récupère le type de requete demandé par l'utilisateur et le chemin du dossier/fichier. Ainsi on lance le script avec le type en argument 1 et le chemin en argument 2, et le script lancera la conversion avec le langage associé.

Pour la version actuelle, le script lance automatiquement la traduction en java, mais grâce a la modularité du code, il est facile d'ajouter un autre langage, en respectant les regles pour le langage.

Langages

Nous avons mis des readmes, dans les dossiers langages, pour permettre de creer d'autres langages

L'extension contient deja le langage Java. L'UML est disponible dans l'onglet UML

Le langage java a pour point d'entré le main.py, qui va analyser le code donné, ou le fichier/dossier. Il va ensuite , selon la fonction appelée, lancer l'analyse des fichiers et pour chaque fichier lu, creer des objets liés aux classes et aux packages.

Une fois toute l'analyse du(es) fichier(s) , le main lancera le passage des objets en UML , tout en respectant la syntaxe de JointJS
